# HT-binned Herwig7 DY ‚Üí NanoGEN

GEN-only (NanoGEN) Drell‚ÄìYan, Herwig7 CH3 shower, reusing the Pythia HT gridpacks.
Bins: `HT-400to600`, `HT-600to800`, `HT-800to1200`, `HT-1200to2500`, `HT-2500toInf`.
Copy-and-swap: each fragment = the inclusive Herwig fragment with only the gridpack
path changed (brief constraint #2: physics blocks stay byte-for-byte identical).

## Layout
```
inputs/
  inclusive_fragment.py   <- YOU PROVIDE (the existing inclusive Herwig DY fragment)
  gridpacks.txt           <- YOU FILL    (bin -> cvmfs .tar.xz path)
Configuration/GenProduction/python/
  Herwig7_DY_<bin>_CH3_cff.py   <- generated by make_fragments.py
production/
  make_fragments.py   swap gridpack path, emit 5 fragments, diff-prove 1 line changed
  verify_merging.sh   read xqcut / parton mult from each gridpack run_card (constraint #4)
  make_configs.sh     adapt McM Sequences cmsDriver -> one *_cfg.py per bin (NanoGEN)
  seed_snippet.py     per-job seeds + unique output name (appended to cfg at runtime)
  run_job.sh          condor payload: cmsenv, cmsRun, xrdcp to EOS
  submit.jdl          parametrized 1-bin JDL
  submit_all.sh       sizing table (NJOBS/bin) + submission driver
```

## Workflow (on lxplus / LPC)
1. Drop the inclusive fragment at `inputs/inclusive_fragment.py`; fill `inputs/gridpacks.txt`.
2. `python3 production/make_fragments.py` ‚Äî generates the 5 fragments, prints diffs.
3. `bash production/verify_merging.sh` ‚Äî confirm xqcut/njetsmax per bin (expect 19 / 4).
4. In `CMSSW_RELEASE/src` with the fragments under `Configuration/GenProduction/python/`
   and `scram b`: paste the McM Sequences command into `make_configs.sh`, then run it.
5. `condor_submit` a single benchmark job for `HT-400to600` and `HT-2500toInf` first
   (acceptance checklist), check per-event CPU + merging accept, then tune NJOBS.
6. `./submit_all.sh`.

## NanoGEN / release decision (BLOCKER ‚Äî verify before producing)
McM Sequences = a GEN-SIM `wmLHEGS` request in **CMSSW_10_2_6** (slc7), GT
`102X_upgrade2018_realistic_v11`, era `Run2_2018`. Two problems for this task:
1. **NanoGEN likely absent in 10_2_6** ‚Äî `nanogen_cff`/`NANOGEN` step arrived ~10_6_X.
2. **slc7 on el9** ‚Äî 10_2_6 jobs must run inside a `cmssw-el7` container on LPC el9.

Run `production/verify_nanogen.sh CMSSW_10_2_6` (and a candidate like
`CMSSW_10_6_28` / `CMSSW_13_0_13`) to decide between:
- **(A) single newer release** with Herwig7 CH3 + NanoGEN ‚Äî simplest; its GT/arch
  replace the 102X ones (GEN-only, so detector conditions are immaterial).
- **(B) two-step**: GEN in 10_2_6 (exact reproduction) ‚Üí NANOGEN in a newer release.

`make_configs.sh` defaults to the verbatim McM GT/era/beamspot/geometry + `LHE,GEN,NANOGEN`;
override `CONDITIONS`/`STEP`/etc. once the release is chosen.
